# License-Review Checklist

Check:
* standalone license file
* repository-level license
* path-specific license
* package/SPDX declaration
* README declaration
* copyright notice
* attribution obligations
* modification notice requirements
* copyleft implications
* source-availability requirements
* mixed-origin content
* copied examples
* no-license repositories
* repository redistribution compatibility

Allowed outcomes:
* `verified_permitted`
* `verified_with_conditions`
* `declaration_only`
* `unresolved`
* `not_permitted`

Clarify:
* A package or README license declaration is evidence, but it may not be a complete redistribution notice.
* Absence of a license normally means redistribution permission is unavailable.
* The repository Apache-2.0 license does not relicense third-party material.
* Unknown attribution must remain unknown.
* This checklist is operational guidance, not legal advice.
