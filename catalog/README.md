# Repository Artifact Catalog

This directory holds the unified YAML catalog linking stable sequential IDs (e.g., `SKILL-0001`, `WORKFLOW-0001`) to their canonical source files. 

## Purpose
The catalog acts as an index of repository capabilities. It decouples artifact identity from underlying filename changes, mapping operational, validated assets into the capability map.

## Structure
- `artifacts.yaml`: The master unified catalog index.
- `ARTIFACT.template.yaml`: The schema for adding new artifacts to the catalog.
